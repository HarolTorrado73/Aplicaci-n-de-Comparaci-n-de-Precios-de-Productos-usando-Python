# Contribuir al proyecto

Gracias por tu interés en contribuir.

## Flujo recomendado

1. Crea una rama desde `main`.
2. Realiza cambios pequeños y enfocados.
3. Documenta cambios funcionales en `README.md` o `docs/`.
4. Abre un Pull Request con contexto y evidencia.

## Estándares mínimos

- Mantener compatibilidad con Python 3.
- No romper el flujo principal de la UI.
- Evitar hardcodear secretos, tokens o credenciales.
- Usar nombres y mensajes claros en commits.

## Checklist para Pull Request

- [ ] El cambio tiene propósito claro.
- [ ] Se actualizó documentación si aplica.
- [ ] Se validó el flujo: producto -> dealer -> precio.
- [ ] No se añadieron secretos ni datos sensibles.

## Reporte de bugs

Incluye:

- Comportamiento actual.
- Comportamiento esperado.
- Pasos para reproducir.
- Evidencia (logs/capturas).

## Convención de commits sugerida

- `feat:` nueva funcionalidad.
- `fix:` corrección de error.
- `docs:` cambios de documentación.
- `refactor:` mejora interna sin cambiar comportamiento.
